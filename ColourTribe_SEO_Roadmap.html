Critical SEO Issues — Fix These First

MUST FIX
CRITICAL

React SPA / Client-Side Rendering (CSR) — Biggest Technical Blocker

ColourTribe is built with React 18 + Vite as a Single Page Application using Client-Side Rendering. When Google's crawler visits the site, it initially sees an empty HTML shell with just a <div id="root"></div>. The crawler must execute JavaScript to see content. Googlebot renders JS, but it's slower, less reliable, and content rendered via JS gets lower indexing priority. This is the #1 reason the site has zero indexed pages.

🔧 Fix Options (Ranked by Impact)
  • Option A (Best): Migrate to Next.js 14 with App Router + SSR/SSG. Every page is pre-rendered as HTML. Google sees full content on first crawl. This is the gold standard for React SEO.
  • Option B: Add React SSR using Vite's SSR mode. Render React to HTML on the server. More complex but keeps Vite.
  • Option C (Quickest win): Add Prerendering with vite-plugin-prerender or React Snap. Pre-render key pages to static HTML at build time. Works for mostly static pages.
  • Option D (Temporary patch): Implement React Helmet for meta tags + submit URLs manually in Google Search Console. Does NOT solve JS rendering but helps Googlebot know pages exist.
# Install prerendering plugin for Vite
npm install --save-dev vite-plugin-prerender

# In vite.config.js:
import prerender from 'vite-plugin-prerender'
plugins: [
  prerender({
    routes: ['/', '/products', '/about', '/contact', '/hotel-uniforms',
             '/chef-uniforms', '/corporate-uniforms', '/housekeeping-uniforms']
  })
]

CRITICAL

Heavy JavaScript Bundle Killing Core Web Vitals

The stack includes React 18 + Three.js + Framer Motion + Zustand + Firebase. This creates a massive JavaScript bundle that results in poor LCP (Largest Contentful Paint), high CLS (Cumulative Layout Shift), and bad FID/INP scores. Google uses Core Web Vitals as a direct ranking signal.

🔧 Optimization Actions
  • Three.js: Load lazily only on sections that actually need 3D. Use React.lazy() + Suspense. Remove Three.js from homepage critical path.
  • Framer Motion: Use LazyMotion with domAnimation feature set (~5KB vs 25KB) instead of full Framer Motion.
  • Code splitting: Use Vite's rollupOptions.output.manualChunks to split vendor bundle.
  • Image optimization: Convert all images to WebP/AVIF format. Use loading="lazy" on below-fold images.
  • Font loading: Use font-display: swap and preload critical fonts.
  • Target: LCP < 2.5s, CLS < 0.1, INP < 200ms
